# CGOS Projection Revision Runtime Versioning Rationale v0.1

`doc_id: CGOS-PROJECTION-REVISION-RUNTIME-VERSIONING-RATIONALE-v0.1`

## A. Why Not Downstream-Aligned Versioning

- downstream product release numbers are not upstream runtime versions
- runtime capability maturity and product release maturity are separate
- upstream versioning must reflect Cognitive_OS capability surface, tests,
  compatibility, and release policy

## B. Why Capability-Line RC Is Safer

- projection revision runtime surface is real and tested
- downstream evidence exists
- but full Cognitive_OS platform/runtime version line is not independently
  sealed
- capability-line RC avoids false platform-level version claim
