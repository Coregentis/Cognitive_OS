# CGOS-DELTA-DRIFT-IMPACT-FIRST-PASS-PLAN-v0.1

## Purpose

This plan freezes what `Delta Drift & Impact first pass` means in current
`Cognitive_OS` repo reality.

It exists because delta/change handling must not be invented from downstream
product pressure, and must not skip over MPLP truth, crosswalk classification,
VSL continuity substrate, or PSG semantic relation substrate.

## Current Repo Reality

`Cognitive_OS` already has:

- a bounded executable fresh-intent path
- a bounded executable requirement-change / delta-intent path
- VSL first-pass continuity state and continuation anchors
- PSG first-pass project-scoped semantic relation state

`Cognitive_OS` does not yet have:

- a full drift engine
- a full impact propagation engine
- a full conflict-resolution engine
- compensation execution
- rollback execution
- SoloCrew exception-plane projection

That means the next lawful move is a bounded runtime-private assessment slice,
not a full recovery or governance-completion wave.

## What Delta Drift & Impact First Pass Means Now

In current repo reality, this wave means:

- fresh intent remains distinct from delta intent
- a delta-intent path may use an existing VSL continuation anchor when one is
  available for the same project
- a delta-intent path may use existing PSG graph state to discover directly
  related affected objects
- drift-record output may be enriched with runtime-private impact fields
- trace/decision evidence may be linked into drift assessment
- conflict-case remains conditional and appears only when explicit reconcile
  tension is present

This wave is still bounded to:

- direct graph-neighbor impact only
- project-scoped runtime-private assessment only
- no compensation execution
- no rollback execution
- no product review queue or founder packet law

## What This Wave Includes

- fresh intent vs delta intent distinction preserved
- continuation anchor usage from VSL when available
- affected object discovery from PSG using direct relation neighbors
- drift-record enrichment with:
  - baseline refs
  - affected refs
  - continuation anchor ref
  - impact summary
  - supporting evidence refs
- deterministic impact summary formation
- conflict-case creation only when explicit tension exists
- reconciliation summary enrichment for runtime-private inspection

## What This Wave Explicitly Excludes

- full impact propagation
- multi-hop semantic traversal
- rollback execution
- compensation execution
- conflict-resolution execution
- SoloCrew exception-plane behavior
- product DTOs
- MPLP top-level object promotion
- protocol export widening

## Why Delta Drift Remains Runtime-Bound / Protocol-Adjacent

Current MPLP truth gives:

- `delta_intent` event-family semantics
- `impact_analysis` and `compensation_plan` event-family semantics
- runtime guidance for drift/rollback patterns

Current MPLP truth does not give:

- a top-level protocol-native `delta-intent` object
- a top-level protocol-native `drift-record` object
- a top-level protocol-native `conflict-case` object

Therefore this wave may enrich Coregentis runtime objects and runtime-private
assessment state, but it may not claim new MPLP object law.

## Crosswalk Gate Compliance

This wave follows the ratified crosswalk by preserving:

- `delta-intent` as protocol-adjacent / runtime-bound
- `drift-record` as Coregentis-private runtime
- `conflict-case` as Coregentis-private runtime

The gate is satisfied only if this implementation:

- reuses frozen MPLP truth without widening it
- reuses existing Coregentis registry/binding truth
- uses VSL and PSG as runtime-private substrates
- keeps export scope unchanged
- does not project directly into SoloCrew

## How This Wave Uses VSL

VSL first pass contributes:

- project-scoped continuity state
- continuation-anchor recovery
- bounded replay / rollback / retention metadata

This wave uses only:

- continuation anchor as grounding context for delta assessment

It does not:

- execute replay
- execute rollback
- widen VSL into storage completion

## How This Wave Uses PSG

PSG first pass contributes:

- project-scoped graph state
- typed node/edge relations
- deterministic graph inspection

This wave uses only:

- direct neighbor discovery from baseline refs and continuation anchor refs

It does not:

- perform multi-hop traversal
- claim a graph database doctrine
- export graph internals

## Relation To Later Waves

Later `AEL` still owns:

- governed activation semantics beyond the current bounded path

Later `Governed Learning` still owns:

- pattern scoring
- consolidation logic over repeated impact/drift shapes

Later SoloCrew work still owns:

- exception-plane projection
- founder-facing review semantics
- downstream packet or queue surfaces

This wave only prepares those later surfaces by making drift/impact assessment
more explicit and evidence-aware inside runtime-private boundaries.

## Boundary Conclusion

`Delta Drift & Impact first pass` is complete only if:

- delta intent remains distinct from fresh intent
- VSL continuation anchor is used when available
- PSG directly related objects can be discovered when available
- drift-record is enriched without widening protocol export scope
- conflict-case remains conditional
- no rollback, compensation, or SoloCrew projection semantics are claimed
