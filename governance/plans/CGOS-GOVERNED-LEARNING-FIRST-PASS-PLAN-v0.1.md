# CGOS-GOVERNED-LEARNING-FIRST-PASS-PLAN-v0.1

## Purpose

This plan freezes what `Governed Learning first pass` means in current
`Cognitive_OS` repo reality.

It exists because learning capture must not be invented from downstream product
pressure, and must not be mistaken for autonomous policy mutation, model
training, or MPLP protocol-law promotion.

The required order remains:

- MPLP truth source
- ratified Coregentis crosswalk / binding interpretation
- bounded runtime-private realization
- only later downstream projection

## Current Repo Reality

`Cognitive_OS` already has:

- a bounded executable minimal cognitive loop
- VSL first-pass continuity substrate
- PSG first-pass semantic relation substrate
- Delta Drift & Impact first pass
- AEL first-pass governed activation assessment
- bounded trace / decision evidence generation
- a first-pass consolidation path that already creates:
  - `learning-candidate`
  - `memory-promotion-record`

`Cognitive_OS` does not yet have:

- a full learning engine
- a full consolidation engine
- autonomous policy mutation
- autonomous preference write-back from learning capture
- model training
- MPLP learning-sample export
- downstream SoloCrew learning/preference projection

That means the lawful next move is a suggestion-only, runtime-private candidate
capture wave.

## What Governed Learning First Pass Means Now

In current repo reality, this wave means:

- learning candidate capture becomes more explicit and evidence-aware
- candidate capture may consume:
  - VSL continuity anchor context
  - PSG relation context
  - Delta Drift / Impact assessment context
  - AEL activation outcome
  - trace / decision evidence
  - drift / conflict artifacts when present
- candidate capture stays bounded to runtime-private hints rather than learned
  law
- future MPLP learning-sample eligibility may be noted, but no automatic export
  occurs

This wave remains bounded to:

- the current minimal-loop baseline
- one runtime-private learning assessment per run
- suggestion-only posture
- no autonomous policy mutation
- no model training
- no product preference law

## What This Wave Includes

- runtime-private governed learning assessment
- learning candidate capture with:
  - source episode refs
  - source evidence refs
  - source object refs
  - optional source relation refs
- reuse pattern hint
- failure pattern hint
- policy suggestion hint
- continuity pattern hint
- activation outcome capture
- drift / conflict capture when present
- future derived learning-sample target metadata
- memory-promotion record linkage when present

## What This Wave Explicitly Excludes

- full learning engine behavior
- autonomous policy mutation
- automatic preference write-back from candidate capture
- model training
- automatic promotion to semantic truth
- automatic MPLP learning-sample export
- SoloCrew learning or preference surfaces
- product DTOs
- MPLP top-level object promotion

## Why Learning / Consolidation Remains Runtime-Private

Current MPLP truth gives:

- protocol-facing learning sample schemas
- learning taxonomy and collection guidance

Current MPLP truth does not give:

- a mandated runtime consolidation engine
- a mandated local candidate scoring engine
- autonomous policy mutation authority
- downstream product preference semantics

Therefore this wave may capture runtime-private learning candidates and note
future export eligibility, but it may not claim new MPLP object law and may not
promote local runtime objects into active protocol truth.

## Relation To MPLP Learning Sample Schemas

MPLP currently owns:

- the shape of learning samples
- frozen sample-family taxonomy such as:
  - `intent_resolution`
  - `delta_impact`
  - `pipeline_outcome`
  - `confirm_decision`
  - `graph_evolution`

This wave may therefore record a future derived sample-family target for later
governed export.

This wave may not:

- export `learning-candidate` directly as an MPLP object
- claim that every runtime candidate is already an MPLP learning sample
- mutate protocol law from repeated local usefulness

## Crosswalk Gate Compliance

This wave follows the ratified crosswalk by preserving:

- `learning-candidate` as Coregentis-private runtime
- consolidation logic as runtime-private
- MPLP learning sample schemas as protocol-side shapes only
- derived export as future-governed and indirect only

The gate is satisfied only if this implementation:

- reuses MPLP truth without widening it
- reuses current Coregentis registry and binding truth
- keeps learning assessment runtime-private
- keeps learning candidate non-exported by default
- does not project directly into SoloCrew

## Runtime Inputs Consumed In This Wave

This wave may consume the following already-lawful runtime inputs:

- VSL continuity state
  - for continuation anchor awareness
- PSG graph state
  - for relation-aware source context
- Delta Drift & Impact assessment
  - for drift / affected-object hints
- AEL assessment
  - for activation outcome and bounded failure/reuse signals
- trace evidence
- decision record
- drift record
- conflict case

## Suggestion-Only Boundary

This wave must preserve:

- `suggestion_only = true`
- `policy_mutation_applied = false`
- `semantic_promotion_applied = false`

That means:

- learning outputs may suggest later review
- learning outputs may not rewrite runtime law automatically
- learning outputs may not become stable semantic truth automatically

## Relation To Later SoloCrew Consumption

Later SoloCrew work may consume projected learning suggestions, review queues,
or preference candidates.

This wave does not create those projection contracts.

It only prepares them by making runtime-private learning assessment explicit and
better evidenced inside `Cognitive_OS`.

## Boundary Conclusion

`Governed Learning first pass` is complete only if:

- candidate capture is evidence-aware
- reuse / failure / policy / continuity hints are explicit
- learning outputs remain runtime-private
- future MPLP sample eligibility is recorded only as indirect metadata
- no autonomous mutation or export occurs
- no SoloCrew learning/preference law is claimed
