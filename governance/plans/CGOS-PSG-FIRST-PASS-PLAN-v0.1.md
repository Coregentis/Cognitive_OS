# CGOS-PSG-FIRST-PASS-PLAN-v0.1

## Purpose

This plan freezes what `PSG first pass` means in current `Cognitive_OS` repo
reality before runtime code claims any semantic relation substrate.

The governing order remains:

- MPLP truth source
- Cognitive_OS crosswalk and binding classification
- bounded runtime realization
- only later downstream projection

This wave is therefore lawful only as a runtime-private substrate wave. It is
not a protocol schema wave, not a product projection wave, and not a full graph
runtime completion wave.

## What PSG First Pass Means Now

`PSG first pass` means:

- a project-scoped runtime-private semantic graph state can exist in
  `Cognitive_OS`
- existing runtime object records can be represented as graph nodes without
  inventing new protocol DTO law
- typed relationship edges can be derived from already-existing runtime fields
  and existing Coregentis relationship vocabulary
- source lineage and evidence references can be preserved in graph-adjacent
  runtime state
- graph inspection can happen deterministically by project
- minimal graph-update summary metadata can exist as runtime-private evidence of
  local graph mutation

This wave remains below:

- full PSG realization
- graph database doctrine
- impact propagation
- drift engine completion
- governed learning graph traversal
- MPLP graph event export
- SoloCrew projection or founder-surface semantics

## What This Wave Includes

- project-scoped graph state
- runtime object node identity
- typed relationship edges drawn from the existing Coregentis relationship
  vocabulary
- source lineage / evidence reference preservation
- graph update summary metadata for runtime inspection
- non-exportability of local PSG internals
- relation to the already-added VSL continuity substrate

## What This Wave Explicitly Excludes

- no MPLP protocol schema widening
- no new DTO family
- no product card, packet, review queue, founder, or Secretary semantics
- no full graph database selection or persistence doctrine
- no full impact analysis engine
- no Delta Drift expansion beyond relation substrate preparation
- no governed learning engine realization
- no claim that local PSG internals are exportable MPLP-native objects

## Why PSG Remains Runtime-Private

Current MPLP truth makes PSG a runtime concept and logical state model, not a
protocol-native top-level schema object.

Therefore this wave may:

- store project-scoped graph state internally
- organize existing runtime objects semantically
- emit only runtime-private graph update summaries

This wave may not:

- treat PSG internals as MPLP law
- export local nodes or edges as canonical protocol objects
- project PSG internals directly into SoloCrew

## Crosswalk Gate Compliance

This plan follows
`CGOS-MPLP-RUNTIME-SEMANTIC-CROSSWALK-AND-VSL-BINDING-RATIFICATION-v0.1` by
keeping PSG classified as `Coregentis-private runtime`.

The gate is satisfied only if this wave:

- consumes existing MPLP truth without widening it
- binds to existing Coregentis runtime objects
- preserves registry/object ownership
- keeps exportability explicitly bounded
- remains product-neutral
- adds tests for project isolation, deterministic inspection, and
  non-exportability

## Relation To VSL

VSL first pass already established project-scoped continuity state,
continuation anchors, and replay / rollback / retention metadata.

PSG first pass is adjacent but distinct:

- VSL first pass preserves continuity substrate
- PSG first pass preserves semantic relation substrate

This wave may read the same project-scoped runtime objects that VSL continuity
checkpointing reads, but it does not turn continuity state into graph law and
does not claim storage completion.

## Why This Wave Is Lawful Now

This repository already has:

- frozen object registry truth
- frozen relationship vocabulary
- frozen binding matrix truth
- a bounded executable minimal loop
- VSL first-pass continuity substrate
- a ratified MPLP-to-runtime crosswalk

That makes a minimal runtime-private PSG substrate lawful now, provided the
implementation stays additive, bounded, and non-exportable.

## What Later Waves Still Own

`Delta Drift` later owns:

- typed impact propagation
- semantic divergence traversal
- compensation and rollback-sensitive graph interpretation

`Governed Learning` later owns:

- consolidation-driven graph promotion
- pattern scoring
- reuse/failure graph semantics

`AEL` later owns:

- governed activation scopes and action propagation rules

This wave only prepares their substrate. It does not implement their behavior.

## Boundary Conclusion

`PSG first pass` is lawful only as:

- project-scoped graph state
- runtime object node identity
- typed relation edges
- lineage/evidence preservation
- deterministic runtime-private inspection

It is not protocol law, not product law, not full graph completion, and not yet
ready for direct SoloCrew consumption.
